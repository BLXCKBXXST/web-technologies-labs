\section{Сетевой шлюз и задача доступа в Интернет}
\label{sec:gateway}

Если в организации имеется локальная сеть, её клиентам необходимо предоставить доступ
в Интернет. Для этого настраивается \textbf{шлюз}~--- узел сети, принимающий запросы
внутренних клиентов и пересылающий их во внешнюю сеть, а входящие ответы
передающий обратно в локальную сеть~\cite{linux_admin}.

В рассматриваемом стенде роль шлюза выполняет виртуальная машина \texttt{gateway}
(Ubuntu~20.04.5~LTS Server) с двумя сетевыми интерфейсами:
\begin{itemize}
  \item \texttt{enp0s3}~--- подключение к Интернету; для VirtualBox~--- адаптер типа
    \textit{NAT} или \textit{Сетевой мост}, получает адрес статически или по DHCP;
  \item \texttt{enp0s8}~--- подключение к локальной сети; адаптер типа
    \textit{Внутренняя сеть}, статический адрес \texttt{192.168.N.1/24},
    где~$N$~--- номер студента в журнале.
\end{itemize}

\section{NAT и механизм MASQUERADE}
\label{sec:nat}

\textbf{NAT} (Network Address Translation)~--- механизм преобразования сетевых адресов
в IP-пакетах при их прохождении через шлюз~\cite{iptables_man}. При использовании
режима \textbf{MASQUERADE} IP-адрес источника в исходящем пакете подменяется
адресом внешнего интерфейса шлюза. Это позволяет множеству внутренних узлов
выходить в Интернет через единственный внешний IP-адрес.

Для маршрутизации IPv4-пакетов ядро Linux использует параметр
\texttt{net.ipv4.ip\_forward}. По умолчанию он отключён (значение~\texttt{0});
для работы шлюза его необходимо включить (значение~\texttt{1}),
прописав в файл \texttt{/etc/sysctl.conf}.

\section{Подсистема netfilter и утилита iptables}
\label{sec:iptables}

\textbf{netfilter}~--- подсистема ядра Linux, выполняющая фильтрацию и
трансформацию сетевых пакетов. \textbf{iptables}~--- пользовательский интерфейс
управления netfilter~\cite{iptables_man}. Правила организованы в таблицы и
цепочки (\textit{chains}). Для реализации NAT используется таблица \texttt{nat}
со следующими основными цепочками:
\begin{itemize}
  \item \texttt{PREROUTING}~--- обработка входящих пакетов \textit{до} принятия
    решения о маршруте (применяется для \texttt{DNAT});
  \item \texttt{POSTROUTING}~--- обработка пакетов \textit{после} принятия
    решения о маршруте, перед отправкой в сеть (применяется для \texttt{MASQUERADE});
  \item цель \texttt{MASQUERADE}~--- подстановка адреса внешнего интерфейса
    вместо исходного адреса источника.
\end{itemize}

Для сохранения правил после перезагрузки используется пакет
\texttt{iptables-persistent}: при старте системы правила автоматически
загружаются из файлов \texttt{/etc/iptables/rules.v4}
и~\texttt{/etc/iptables/rules.v6}.

\section{Протокол DHCP}
\label{sec:dhcp}

\textbf{DHCP} (Dynamic Host Configuration Protocol, RFC~2131)~\cite{rfc2131}~---
протокол автоматической настройки сетевых параметров клиента: IP-адреса, маски
подсети, адреса шлюза и DNS-серверов. Обмен сообщениями происходит в четыре
этапа:
\begin{enumerate}
  \item клиент рассылает широковещательный запрос \texttt{DHCPDISCOVER};
  \item сервер отвечает предложением \texttt{DHCPOFFER};
  \item клиент подтверждает выбор сервера сообщением \texttt{DHCPREQUEST};
  \item сервер подтверждает выдачу адреса сообщением \texttt{DHCPACK}.
\end{enumerate}

Сервер \texttt{isc-dhcp-server}~--- наиболее распространённая реализация DHCP
для Linux-систем от Internet Systems Consortium~\cite{isc_dhcp}. Конфигурация
хранится в \texttt{/etc/dhcp/dhcpd.conf} и включает описание подсети, диапазона
выдаваемых адресов, адресов шлюза и DNS, а также времени аренды (lease time).
При необходимости можно выполнить резервирование~--- закрепить конкретный
IP-адрес за устройством с заданным MAC-адресом.

\section{Утилита Netplan}
\label{sec:netplan}

\textbf{Netplan}~--- утилита настройки сетевых интерфейсов, применяемая по
умолчанию в Ubuntu, начиная с версии~17.10~\cite{netplan_docs}. Конфигурация
задаётся в YAML-файле \texttt{/etc/netplan/*.yaml} и применяется командой
\texttt{netplan apply}. Важные особенности синтаксиса YAML:
\begin{itemize}
  \item для отступов используются \textit{только пробелы} (символы табуляции
    недопустимы);
  \item значение параметра записывается после двоеточия через пробел;
  \item вложенность определяет принадлежность параметров к объекту.
\end{itemize}
