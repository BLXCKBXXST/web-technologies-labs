\section{Теоретические сведения}

\subsection{Система доменных имён DNS}

DNS (Domain Name System)~--- иерархическая распределённая система именования узлов сети~\cite{rfc1034}. Пространство имён имеет древовидную структуру: корень, домены верхнего уровня (TLD), домены второго уровня и т.д.

Основные понятия:
\begin{itemize}
  \item \textbf{Зона}~--- административная единица пространства имён;
  \item \textbf{Авторитативный сервер}~--- хранит оригинальные записи зоны;
  \item \textbf{Рекурсивный резолвер}~--- выполняет цепочку запросов от имени клиента;
  \item \textbf{Форвардер}~--- проксирует запросы для нелокальных зон на внешний сервер.
\end{itemize}

\subsection{Прямая и обратная зоны}

\textbf{Прямая зона} (forward zone) используется для разрешения имён в IP-адреса (записи SOA, NS, A). \textbf{Обратная зона} (reverse zone) строится инверсией октетов с суффиксом \texttt{.in-addr.arpa} (записи SOA, NS, PTR). Например, для сети \texttt{192.168.N.0/24} зона~--- \texttt{N.168.192.in-addr.arpa}.

\subsection{BIND9}

BIND~9~--- наиболее распространённое ПО DNS-сервера~\cite{bind9_admin}. Конфигурационные файлы: \texttt{/etc/bind/named.conf}, \texttt{named.conf.options}, \texttt{named.conf.local}. Файлы зон хранятся в \texttt{/var/lib/bind/} (BIND имеет права на запись, что необходимо для DDNS).

\subsection{Динамическое обновление DNS (DDNS)}

DDNS (RFC~2136)~\cite{rfc2136} позволяет DHCP-серверу автоматически создавать A- и PTR-записи при выдаче IP-адреса клиенту. Для аутентификации используется TSIG-ключ \texttt{rndc-key} (HMAC-MD5), хранящийся в \texttt{/etc/bind/rndc.key}.
