\chapter*{Заключение}
\addcontentsline{toc}{chapter}{Заключение}
\label{ch:conclusion}

В ходе выполнения лабораторной работы~\cfgLabNumber{} был развёрнут и настроен
DNS-сервер \textbf{BIND9} на виртуальной машине \texttt{gateway}
под управлением Ubuntu~20.04~LTS.

Выполненные задачи:
\begin{itemize}
  \item присвоено имя хосту сервера (\texttt{hostnamectl set-hostname});
  \item удалены устаревшие DNAT-правила переадресации DNS из лаб.~работы~4;
  \item установлены и настроены пакеты \texttt{bind9} и \texttt{dnsutils};
  \item создана прямая зона \texttt{STUDENT.GROUP.local} с записями SOA, NS, A;
  \item создана обратная зона
        \texttt{\cfgLabStudentN.168.192.in-addr.arpa} с записями PTR;
  \item настроен форвардер на \texttt{8.8.8.8} для внешних запросов;
  \item настроено DDNS с ключом \texttt{rndc-key} для автоматической
        регистрации DHCP-клиентов в DNS;
  \item проверена работа DNS командами \texttt{nslookup} и
        \texttt{tail /var/log/syslog}.
\end{itemize}

Полученные навыки --- администрирование BIND9, работа с файлами зон,
настройка форвардеров и DDNS --- являются базовыми компетенциями
сетевого администратора и широко применяются при построении корпоративных сетей.
