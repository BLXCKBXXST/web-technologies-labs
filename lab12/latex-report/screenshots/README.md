## 🖼️ Как делать скриншоты

Два способа в зависимости от типа скриншота:

### Скриншоты 01–04 (локальные рендеры)

Сначала открыть локальную страницу:

```bash
xdg-open ../../codepen-solutions/1.1-html-tags/index.html
# и так для остальных трёх блоков
```

Затем снять скриншот окна браузера:

- **Linux:** `flameshot gui` (выделить мышью область) либо
  `gnome-screenshot -w` (активное окно целиком);
- **Windows:** `Win + Shift + S`;
- **macOS:** `Cmd + Shift + 4`, затем пробел → клик по окну.

### Скриншоты 05–09 (Codepen)

Залогиниться в Codepen, открыть свой форк pen-а, сделать видимыми
нужные панели (для блоков 1.1 и 1.2 — HTML и Preview; для блоков 2.1
и 2.2 — HTML, CSS и Preview) через кнопки `Layout` в нижней панели
Codepen. Затем — те же инструменты, что и для скриншотов 01–04.

Скриншот `09_codepen_save.png` — увеличенный фрагмент правой верхней
панели любого pen-а с видимыми кнопками `Fork` и `Save`.

## ⚠️ Важно

- Имена файлов должны совпадать в трёх местах: эта таблица,
  `\includegraphics{...}` в `parts/chap2.tex` / `parts/chap3.tex`,
  и сами файлы в `../img/`.
- Перед снятием скриншотов 01–04 убедиться, что страница полностью
  отрендерилась: внешние стили Netology и Material Design Lite
  тянутся с CDN, и при первом открытии без интернета список и таблицы
  будут выглядеть как голые HTML-теги.
- Для блока 1.1 на скриншоте `01` должны быть видны все три
  фотографии гейзеров. Если страница длинная — снять либо в полный
  размер окна (предварительно растянув), либо склеить из двух
  скриншотов.
- Для блока 2.2 на скриншоте `04` должна быть видна цитата с
  фоновой картинкой — она находится в самом низу страницы.
