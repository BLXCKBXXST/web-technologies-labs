# Практическая работа №14
**Скачивание файла в формате PDF. Эффект «Material Wave»**

Интерактивное веб-резюме на чистых HTML, CSS и JavaScript — без библиотек, фреймворков и сборки. Текст резюме редактируется прямо на странице, правки сохраняются в браузере, готовый документ выгружается в PDF. Каждое изменение анимировано, клики дают эффект «волны» Material Design.

---

## 🛠️ Среда

| Что | Где |
|---|---|
| Браузер | Chrome/Firefox/Safari — открывается напрямую (`file://`) или с веб-сервера |
| Линтеры JS | [JSHint](https://jshint.com), [ESLint](https://eslint.org) — проверка качества кода |
| Валидатор разметки | [validator.w3.org](https://validator.w3.org) — проверка семантики вёрстки |
| Хостинг | домашний сервер `server34.netcraze.club` + reverse-proxy [Caddy](https://caddyserver.com/) |

---

## 📋 Задание

Согласно [методичке](Методичка%20(lab14).pdf):

1. **Вёрстка резюме** по типовому шаблону — семантическая разметка HTML5, две колонки на CSS Grid.
2. **Редактирование «на месте»** — текстовые элементы правятся через `contenteditable`, изменения сохраняются внутри элементов и в `localStorage`.
3. **Кнопка «Скачать»** — экспорт текущей версии резюме в PDF средствами `window.print()` и стилей `@media print`.
4. **CSS-анимации** — каждое изменение элемента анимировано (`@keyframes`).
5. **Material Wave** — эффект «волны» (ripple) при кликах по элементам.
6. **Адаптивность** и **сохранение данных** при перезагрузке страницы.

Всё реализовано базовыми технологиями — без сторонних библиотек и фреймворков.

## 🚀 Запуск

Сборки нет — это набор статических файлов. Достаточно открыть страницу:

```bash
xdg-open resume/index.html
```

Либо поднять локальный HTTP-сервер (необязательно, но ближе к продакшену):

```bash
cd resume && python3 -m http.server 8000
# открыть http://localhost:8000
```

### Как пользоваться

- **Редактировать текст** — кликнуть по любому полю резюме и править прямо на странице.
- **Загрузить фото** — кликнуть по аватару и выбрать файл изображения.
- **Скачать PDF** — кнопка «Скачать PDF» открывает диалог печати; выбрать принтер «Сохранить как PDF».
- **Сбросить** — кнопка «Сбросить» очищает правки и возвращает исходный шаблон.

## 🔒 Данные и приватность

В репозиторий закоммичен **выдуманный персонаж** (фронтенд-разработчик с вымышленными ФИО и контактами в `config.js`). Реальное резюме заполняется **в браузере**: текст — кликом по полям, фото — кликом по аватару. Все реальные данные и фотография хранятся **только в `localStorage` браузера** и ни в один файл проекта не записываются — в публичный репозиторий они попасть не могут. Кнопка «Сбросить» полностью стирает их.

## 🌐 Хостинг

Развёрнутая версия (временно, на период сдачи задания): **<https://resume.server34.netcraze.club>**

Деплой и снос делаются скриптом [deploy.sh](deploy.sh):

```bash
./deploy.sh --install     # развернуть сайт
./deploy.sh --uninstall   # снести сайт после демонстрации
```

`--install` идемпотентно: rsync исходников из [resume/](resume/) в `/opt/stack/resume/site`, добавляет сервис `resume` (`nginx:alpine`) в общий `/opt/stack/docker-compose.yml`, дописывает блок поддомена в `Caddyfile` и перезагружает Caddy. TLS-сертификат выпускается автоматически Let's Encrypt при первом обращении. `--uninstall` выполняет обратные действия. Скрипт требует уже установленного Caddy (см. [home-server/scripts/50-install-caddy-proxy.sh](https://github.com/BLXCKBXXST/home-server/blob/main/scripts/50-install-caddy-proxy.sh)).

На сервер уезжает резюме с выдуманным персонажем — личные данные остаются в браузере.
